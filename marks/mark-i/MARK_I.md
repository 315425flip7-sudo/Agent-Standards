# Mark I — System Genesis (Historical Excerpt)

> **Status:** Superseded by Mark II. Preserved here as the lineage record.
> This is an *excerpt* — a summary + artifact index, not the full original text.

## What Mark I Was
The **Minimum Viable Frame (MVF)** for Hermes agent initialization — the first attempt to
codify a general operational policy that any newly spawned agent could ingest. It established:

- **The 3-Turn Operational Protocol** — PLAN / EXECUTE / DEBRIEF, with a 4-point AAR at close.
- **The Trinity** as a universal analytical lens (Soul / Mind / Body).
- **The bootstrap-via-repo idea** — a new agent clones the `Agent-Standards` repo, saves the
  injected rules to memory, and copies the skill frameworks into `~/.hermes/skills/`.
- **Core Policy vs. Local Culture** — the repo carries universal rules; each agent grows its own culture on top.

## Why It Was Superseded
Mark I proved the bootstrap mechanism but lacked lexical precision and a clean Bible→Books→Marks
cosmology. Mark II corrects the structure (one repo = one Book), locks the canonical frameworks,
and formalizes the deadzone exception to the 3-Turn Protocol.
